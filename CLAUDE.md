# linux-kernel-security-oss(公开仓)— 项目上下文

> Claude Code 打开本目录即加载。**这是公开开源仓,面向任何人 clone。铁律:零真实产品/厂商/客户信息。**

## 这是什么

给 Linux vendor / Android arm64 内核做**防御性安全检测**的开源工具编排:分类六件事(已知 CVE / 配置加固 / 新洞挖掘 / 影响评级 / LLM 审计 / 二进制逆向)+ 一键装依赖 + `bin/ksec` 三层检测。工具本身是第三方开源件,本库负责编排和上手。

## 铁律:公开仓零敏感信息

- **禁止**出现任何真实芯片型号 / 厂商名 / 机型 / 客户名 / 内部路径 / 未披露漏洞指纹。
- 所有示例一律虚构:驱动 `foo` / `/dev/foo` / `foo.ko` / `FOO_IOC_*` / `0xNNNN`;内核路径用 `$KERNEL`;SoC 用 `SoC-X`。
- 真实案例(带真实平台值的漏洞测试)**只存在于配套的内部私有仓**,永不进本仓库。本仓库是"工具+方法",内部仓是"拿真目标实测 + 反向验证本仓库设计"。
- **改任何文件后、push 前**:跑脱敏 grep,产品名/厂商名/内部路径 0 命中才算过。

## 双仓拓扑(记住职责边界)

| | 本仓(公开) | 配套内部仓(私有,不在此) |

| git 历史 | 干净,零产品名(连 commit message) | 私有,可含真值 |
| 远程 | 可 push 公开(GitHub) | 无公开远程 |
| 信息流 | 内部仓的设计反馈**脱敏后**才回灌到这 | 消费本仓工具打真目标 |

**信息只能从内部 → 公开 时经脱敏闸;真名/指纹永不进本仓库。**

## 定位:防御性

- 实践 ①②③⑤⑥(检测 / 加固 / 挖洞→修 / 审计 / 逆向确认),对自有或授权目标做,目的是**发现并修复**。
- ④ 是参考类:只做纸面影响评级给洞定优先级,**不放攻击工具本体、不写利用链**。描述统一用检测/加固/影响评级/修复口径。

## 约定

- `bin/ksec` 只用 Python3 标准库;各扫描器缺了优雅跳过(报告标 skipped)。脚本开头锁 `PATH` + `PYTHONNOUSERSITE=1`。
- `repos/` 第三方克隆字节不进 git(`.gitignore`),靠 `scripts/clone-tools.sh` 重建。
- **commit message 禁止任何 Claude / Anthropic 署名。**
- 临时文件不用 `/tmp`;路径用 `ROOT`/`$KERNEL` 相对,不写死机主目录。
- 设计 HTML(若加介绍页)走 anthropic-design skill + 发布前三闸。
